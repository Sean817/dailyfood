## 📋 测试步骤

### 测试1: 基本界面检查
1. ✅ 打开 http://localhost:5173
2. ✅ 检查页面标题是否为"孕妇饮食记录 - Daily Food"
3. ✅ 检查是否显示：
   - 顶部标题："🤰 孕妇饮食记录"
   - 日期选择器
   - "添加食物"表单（左侧）
   - "食物清单"区域（右侧）
   - "营养分析"面板（底部）

### 测试2: 添加食物 - 搜索功能
1. ✅ 在"添加食物"表单的"食物名称"输入框中输入"鸡"
2. ✅ 应该看到下拉菜单显示搜索结果（如：鸡胸肉、鸡蛋等）
3. ✅ 点击"鸡胸肉"选项，名称自动填入
4. ✅ 输入数量：100
5. ✅ 选择餐次：早餐
6. ✅ 点击"添加"按钮
7. ✅ 检查食物是否出现在右侧的"早餐"列表中

### 测试3: 添加多种食物
按以下顺序添加食物：

**早餐：**
- 鸡蛋，100g
- 全麦面包，50g

**午餐：**
- 米饭，200g
- 鸡胸肉，150g
- 西兰花，100g

**晚餐：**
- 面条，150g
- 瘦猪肉，100g
- 菠菜，150g

**加餐：**
- 苹果，200g
- 酸奶，100g

### 测试4: 营养分析验证
添加上述食物后，检查营养分析面板：

1. ✅ **热量**：应该显示总热量和进度条
2. ✅ **蛋白质**：检查蛋白质摄入量
3. ✅ **钙**：检查钙摄入量（应该较高，因为有牛奶、酸奶、菠菜等）
4. ✅ **铁**：检查铁摄入量（菠菜含铁高）
5. ✅ **叶酸**：检查叶酸摄入量（菠菜、西兰花含叶酸高）
6. ✅ **进度条颜色**：
   - 绿色：达标（≥80%）
   - 橙色：不足（<80%）
   - 红色：超标（>100%）

### 测试5: 删除功能
1. ✅ 在食物列表中，找到任意一个食物项
2. ✅ 点击右侧的删除图标（垃圾桶图标）
3. ✅ 检查食物是否从列表中消失
4. ✅ 检查营养分析是否相应更新

### 测试6: 日期切换
1. ✅ 点击日期选择器
2. ✅ 选择不同的日期（如：昨天或明天）
3. ✅ 检查食物列表是否为空（新日期）
4. ✅ 在新日期添加食物
5. ✅ 切换回原日期，检查之前的记录是否还在

### 测试7: 数据持久化
1. ✅ 添加一些食物记录
2. ✅ 刷新浏览器页面（F5 或 Cmd+R）

### 测试8: 响应式设计

## 🐛 常见问题排查

### 问题1: 页面无法打开
- 检查开发服务器是否运行：`lsof -ti:5173`
- 确认地址为：http://localhost:5173

### 问题2: 搜索没有结果
- 确认输入了正确的食物名称
- 检查食物数据库中是否包含该食物（查看 `src/data/foodDatabase.ts`）

### 问题3: 营养计算不正确
- 检查输入的数量单位是否为"克"
- 确认食物名称与数据库中的名称完全匹配

### 问题4: 数据丢失
- 检查浏览器是否允许 localStorage
- 尝试清除浏览器缓存后重新测试
